# CMP2 项目章程

## 核心原则
### I. 权威数据源与一致性
系统明确“权威数据源”归属（SIM 状态/用量/话单/资费）；所有同步与回写遵守一致性与幂等；对账、计费与展示以权威源为准。

### II. 租户隔离与最小权限
严格按“系统管理员 / 代理商组织 / 企业组织 / 部门”分层隔离；默认最小可见与最小可操作，跨层访问须明确授权。

### III. 状态机与生命周期不可违背
SIM 必须遵守 INVENTORY → TEST_READY → ACTIVATED → DEACTIVATED → RETIRED 状态机；禁止非法迁移；所有变更需权限校验与审计。

### IV. 审计与可追溯性
组织/权限、SIM 生命周期、资费与订阅、导入导出、账单与对账等关键操作必须记录不可篡改审计日志。

### V. 上游集成可靠性与幂等
与供应商 CMP 的交互必须具备幂等、重试与回执确认；失败与重试可观测且可追踪。

## 技术栈与附加约束
- 前端：原生 HTML + CSS + JavaScript
- 性能与可用性：核心接口 P95 ≤ 300ms；可用性 ≥ 99.9%
- 数据安全：供应商主数据加密存储并保留历史变更记录
- 完整性：运营商 MCC/MNC 校验有效；关联变更触发同步或重算

## 开发流程
- 需求与验收：状态机、权限边界、对账链路须有明确验收口径
- 变更控制：涉及租户模型、对账链路或状态机的变更需评审与迁移计划
- 质量门禁：核心流程必须覆盖测试（租户隔离、状态迁移、幂等/重试）

## 治理
- 章程优先于其他实践，冲突以章程为准
- 变更需文档化与批准，并提供迁移计划
- 所有评审需验证合规性与复杂性合理性
